Why Riot?

  • Lightweight - several times smaller than most libraries
  • Simple - it’s just javascript, html, and css
  • Plays well with others
  • Actively developed (v4.0 coming soon)
  • Very stable

Why not Riot?

  • No two way binding, state, or routing (but you can add it)
  • Maybe too lightweight for some
  • Community is not as large as others
  • Not as many 3rd party libraries

Expressions (template syntax)

  • 100% javascript
  • Single braces (can be changed to pretty much anything)
  • Scoped: this -> parent tag -> ... -> window
  • Strings are HTML-escaped (to avoid XSS)
  • HTML attributes evaluate with special short-hand syntax

Tag Context Variables

  • In a tag, this.opts is an object passed in when a tab is mounted
  • Attributes on the tag overwrite values here

Lifecycle Events

  • Custom JS executed at every stage of a tags existence
  • before-mount/mount - When riot.mount is called
  • update/updated - After mount and any time an internal event happens
  • before-unmount/unmount - When the tag is destroyed

Mixins

  • Re-use code across tags
